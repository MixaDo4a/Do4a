# User Guide

## Вход

Локальные тестовые пользователи:

- `manager@do4a.local` - менеджер.
- `auditor@do4a.local` - проверяющий.
- `store.manager@do4a.local` - управляющий.
- `super.admin@do4a.local` - супер-админ.
- `developer@do4a.local` - разработчик.

Пароль для локального теста:

```text
Do4aTest345
```

В рабочей версии основной вход будет через Telegram Mini App, email останется резервным способом.

## Как создать учетную запись

Технический порядок:

1. Создать пользователя в Supabase Auth.
2. Создать или выбрать сотрудника в `employees`.
3. Создать профиль в `profiles`, связав `auth.users.id` с `employee_id`.
4. Назначить роль в `user_roles`.

Правила:

- Управляющий может создавать сотрудников только с ролью менеджера.
- Роли выше менеджера назначает только супер-админ.
- Статус `Падаван` или `Бывалый` не является ролью и влияет только на расчет чек-листа.

## Менеджер

### Смены

1. Открывает смену.
2. Закрывает смену.
3. При закрытии вводит:
   - наличную выручку;
   - безналичную выручку;
   - возвраты;
   - количество чеков;
   - количество товаров;
   - выемку;
   - комментарий к РКО;
   - аванс;
   - покупюрник.
4. После закрытия система автоматически считает:
   - оборот;
   - средний чек;
   - глубину чека;
   - 2% основному продавцу;
   - 1% второму продавцу;
   - snapshot смены.

### Задачи

Менеджер видит свои задачи, может:

- выполнить задачу;
- пометить задачу как невыполненную;
- оставить комментарий.

## Проверяющий

### Чек-листы

1. Открывает раздел `Чек`.
2. Выбирает магазин.
3. Выбирает сотрудника.
4. Ставит оценку от 1 до 10 по каждому пункту.
5. При необходимости оставляет комментарий.
6. Сохраняет чек-лист.

Система автоматически считает:

- средний балл;
- результат каждого пункта;
- окладную часть за смену.

Формула пункта:

```text
результат = вес пункта / 10 * оценка
```

Если чек-листов за месяц несколько, для зарплаты используется средний результат.

Если чек-листов за месяц нет, используется максимальный результат для статуса сотрудника.

## Управляющий

Может:

- создавать задачи;
- контролировать выполнение задач;
- смотреть смены;
- проводить расчет зарплаты;
- вводить просрочку;
- вводить инвентаризацию;
- вводить товар под ЗП;
- смотреть уведомления;
- управлять сотрудниками и графиком, когда эти экраны будут добавлены.

## Зарплата

Раздел `ЗП` показывает расшифровку:

- смены;
- оборот;
- продажная часть;
- надбавка за выполнение плана;
- оклад по чек-листам;
- премии/штрафы;
- авансы;
- просрочка;
- инвентаризация;
- товар под ЗП;
- итог к выплате.

Формула:

```text
итого = продажи + надбавка за план + оклад + премия - аванс - просрок - инвента - под ЗП
```

## Уведомления

В системе должны быть уведомления:

- новая задача;
- изменение графика;
- напоминание закрыть смену;
- напоминание о смене за 16 часов;
- дедлайн задачи за 30 минут;
- незакрытая смена;
- просроченная задача;
- автозакрытая смена;
- плохой чек-лист.

Каналы:

- внутри приложения;
- Telegram;
- email как резерв.

## Что делать после запуска Docker

1. Запустить Docker Desktop.
2. Проверить Supabase:

```powershell
npx.cmd supabase status
```

3. Применить миграции:

```powershell
npx.cmd supabase migration up --local
```

4. Проверить приложение:

```powershell
npm.cmd run typecheck
npm.cmd run build
```

5. Открыть:

```text
http://localhost:3000
```
