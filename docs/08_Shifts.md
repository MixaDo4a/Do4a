# Shifts

## Назначение

Документ описывает смены как центральную сущность системы.

## Создание и открытие смены

График смен могут создавать:

- управляющий;
- супер-администратор.

Менеджер может открыть смену без заранее созданной записи в графике.

Если смена открыта без графика, система должна создать фактическую смену и пометить источник как manual_open.

## Жизненный цикл смены

Состояния:

- planned - смена запланирована.
- opened - смена открыта основным продавцом.
- closed - смена закрыта вручную основным продавцом.
- auto_closed - смена закрыта автоматически.
- cancelled - смена отменена.
- correction_required - смена требует проверки или дополнения данных.

## Участники смены

В смене может быть:

- основной продавец;
- второй продавец.

Смену открывает и закрывает основной продавец.

Второму продавцу не нужно открывать или закрывать смену.

## Оборот при двух продавцах

Оборот смены идет сразу двум продавцам.

Если магазин наторговал 100, основной продавец получает продажную часть 2, второй продавец получает продажную часть 1.

То есть:

```text
основной продавец = оборот смены * 0.02
второй продавец = оборот смены * 0.01
```

## Закрытие смены

При закрытии смены основной продавец заполняет кассовые показатели:

- дата;
- выручка наличными;
- выручка безналичными;
- возвраты наличными;
- возвраты безналичными;
- количество чеков;
- итоговая выручка;
- инкассация или выемка;
- комментарий к выемке / РКО;
- основной продавец;
- второй продавец, если был;
- длина чека;
- аванс, если был;
- причина выемки или РКО текстом;
- покупюрник: количество купюр и монет по номиналам;
- фото кассового отчета из ККМ.

## Автоматическое закрытие

Если сотрудник забыл закрыть смену, система закрывает ее автоматически в 23:59 по локальному времени магазина.

Автоматически закрытая смена должна получить пометку, что она требует проверки и дополнения данных.

Супер-администратор может позже дополнить и исправить данные такой смены через корректировку.

## Исправление закрытой смены

Исправить закрытую смену может только супер-администратор.

Исправление не должно переписывать историю молча. Должна быть создана корректировка с причиной, автором и audit log.

## Snapshots

После закрытия смены формируется snapshot, который используется для KPI, payroll и аналитики.

Если смена закрыта автоматически и требует проверки, snapshot должен иметь соответствующую пометку качества данных.

## Требует уточнения

- Какие поля аванса нужны при закрытии смены.
- Нужно ли подтверждение закрытия смены управляющим.

## Авансы в смене

Аванс при закрытии смены указывается одной суммой.

Аванс считается взятым тем сотрудником, который закрывает смену и заполняет кассовый отчет.

Аванс может взять только сотрудник, который работает в эту смену.

## Возвраты и оборот

Возвраты фиксируются в кассовом отчете, но не уменьшают оборот сотрудника для расчета продажной части зарплаты.

Оборот для зарплаты считается без вычитания возвратов.

## Глубина чека

Длина чека в текущей таблице означает глубину чека.

Глубина чека считается по формуле:

```text
глубина чека = количество проданных товаров за смену / количество чеков за смену
```

Количество чеков используется для KPI среднего чека и глубины чека.

## Покупюрник

При закрытии смены сотрудник должен заполнить покупюрник.

Сотрудник вводит количество купюр и монет по номиналам. Система автоматически считает:

- сумму по каждому номиналу;
- итоговую сумму в кассе;
- итоговую сумму с мелочью.

## Доступ к фото ККМ

Фото кассового отчета ККМ могут смотреть все пользователи, у которых есть доступ к смене или соответствующему магазину.


## Уведомления по сменам

Менеджер получает напоминание о будущей смене за 16 часов до начала смены.

Менеджер получает напоминание закрыть смену за 10 минут до автоматического закрытия.

Управляющий получает уведомление о незакрытой смене и об автозакрытой смене.
