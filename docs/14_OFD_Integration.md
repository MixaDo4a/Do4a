# OFD Integration

## Назначение

Документ описывает интеграцию с СБИС OFD.

## Основные требования

Система должна получать и хранить операции продаж, возвраты, даты и время операций, кассу, магазин, суммы и идентификаторы операций для защиты от дублей.

## Правила

- OFD данные сохраняются в PostgreSQL.
- Повторный импорт не должен создавать дубли.
- OFD не заменяет внутренние snapshots смен.

## Требует уточнения

- Доступные API СБИС OFD.
- Частота синхронизации.
- Формат данных.
- Правила обработки ошибок.
