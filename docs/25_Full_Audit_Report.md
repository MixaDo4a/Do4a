## OFD-интеграция

Цель интеграции: убрать ручной ввод кассовых данных продавцом.

Минимальная схема:

1. Хранить настройки OFD по магазину и кассе:
   - провайдер;
   - идентификатор кассы;
   - токен/ключ доступа;
   - статус интеграции.
2. Создать таблицу `ofd_daily_reports`.
3. Cron job или Edge Function запрашивает данные OFD по кассе.
4. Система сопоставляет отчет OFD со сменой по магазину и дате.
5. При закрытии смены продавец вводит только:
   - аванс;
   - комментарий к выемке;
   - покупюрник, если нужен;
   - фото, если требуется внутренним контролем.
6. Выручка, возвраты, количество чеков и товары подтягиваются автоматически.
7. Если OFD недоступен, смена закрывается вручную с пометкой `requires_review`.

Нужно получить от СБИС/ОФД:

- актуальную документацию API;
- способ авторизации;
- список доступных методов по кассовым отчетам;
- ограничения по частоте запросов;
- формат данных по возвратам, чекам и позициям.
