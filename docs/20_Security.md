# Security

## Назначение

Документ описывает требования безопасности.

## Основные принципы

- Клиент не доверенная среда.
- Все критичные действия проверяются сервером.
- Используются RLS-политики Supabase.
- Секреты не хранятся во frontend-коде.
- Доступ к данным ограничивается ролью и областью ответственности.
- Критичные изменения логируются.

## Требует уточнения

- Модель ролей в RLS.
- Хранение Telegram user id.
- Политика сессий.
- Права для service role.
