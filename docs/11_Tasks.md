# Tasks

## Назначение

Документ описывает задачи и повторяющиеся задачи.

## Кто создает задачи

Создавать задачи могут:

- управляющий;
- супер-администратор.

## Назначение задачи

Задача ставится на:

- магазин;
- сотрудника.

Задача должна иметь и магазин, и ответственного сотрудника.

## Поля задачи

Задача должна иметь:

- название;
- описание;
- срок выполнения;
- приоритет;
- фото, если нужно;
- комментарии;
- статус;
- автора;
- ответственного сотрудника;
- магазин.

## Повторяющиеся задачи

Задача может быть повторяющейся.

Если предыдущая повторяющаяся задача не выполнена, новая задача все равно создается.

При этом управляющему должно быть отправлено уведомление, что предыдущая задача не выполнена.

## Закрытие задачи

Закрыть задачу могут:

- исполнитель;
- управляющий;
- супер-администратор.

Закрытие задачи должно фиксировать автора закрытия и время закрытия.

## Фотоотчет

По задаче должна быть возможность прикрепить фотоотчет.

Фотоотчет не является обязательным для всех задач.

## Влияние на KPI и зарплату

На старте задачи используются только для контроля выполнения.

Задачи не влияют на KPI и зарплату, пока не будет утверждено отдельное бизнес-правило.

## Правила

- Каждая задача должна иметь магазин и ответственного сотрудника.
- Повторяющаяся задача создается по расписанию даже при невыполненной предыдущей.
- Если предыдущая повторяющаяся задача не выполнена, управляющий получает уведомление.
- Фотоотчет по задаче возможен, но не обязателен по умолчанию.
- Задачи не влияют на payroll и KPI на старте.


## Напоминания по задачам

Исполнитель получает напоминание о дедлайне задачи за 30 минут до срока выполнения.

Если задача просрочена, управляющий получает уведомление.
