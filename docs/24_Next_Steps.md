# Next Steps

## Цель ближайшего этапа

Перевести общее ТЗ в технический фундамент, достаточный для начала разработки приложения.

## Этап 1. Уточнение бизнес-ядра

1. Подтвердить роли и области доступа.
2. Подтвердить жизненный цикл смены.
3. Подтвердить правила второго продавца.
4. Подтвердить формулы KPI.
5. Подтвердить формулы зарплаты.
6. Подтвердить правила ручной кассы.

## Этап 2. Проектирование базы

1. Расширить 02_Database_Design.md до полноценной схемы таблиц.
2. Описать связи между магазинами, сотрудниками, сменами, задачами, KPI и payroll.
3. Отдельно описать snapshots и audit log.
4. Подготовить первую Supabase migration.

## Этап 3. MVP приложения

MVP должен включать:

- авторизацию;
- роли;
- магазины;
- сотрудников;
- график;
- открытие и закрытие смены;
- чеклист открытия;
- задачи;
- базовый dashboard;
- audit log для критичных действий.

KPI, payroll и OFD лучше проектировать сразу, но внедрять после фиксации формул.

## Этап 4. Frontend

2. Настроить TailwindCSS.
3. Сделать mobile-first shell приложения.
4. Добавить экраны по ролям.
