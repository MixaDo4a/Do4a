# Coding Standards

## Назначение

Документ описывает стандарты разработки.

## Общие правила

- TypeScript обязателен.
- Не использовать any без явной причины.
- Бизнес-логику не дублировать между frontend и backend.
- Финансовые расчеты держать в проверяемых функциях.
- Имена сущностей должны совпадать с документацией.
- Изменение бизнес-логики требует обновления документации.

## Frontend

- Next.js App Router.
- Mobile First.
- TailwindCSS.
- Zustand только для клиентского состояния, не как источник истины.

## Backend

- Supabase migrations для схемы.
- RLS для защиты таблиц.
- Edge Functions или RPC для критичных операций.
