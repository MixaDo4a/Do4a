# Authentication

## Назначение

Документ описывает вход пользователей и связь авторизации с сотрудниками.

## Базовый подход

Основной способ входа сотрудников: Telegram.

Запасной способ входа: email. Он нужен на случай ограничений, блокировок или недоступности Telegram.

Система должна поддерживать:

- вход через Telegram Mini App;
- вход через web-приложение по email;
- связь учетной записи пользователя с сотрудником;
- блокировку доступа без удаления истории;
- сохранение истории после увольнения сотрудника.

## Принципы

- Пользователь и сотрудник являются разными сущностями.
- Заблокированный пользователь не может входить в систему.
- Уволенный сотрудник сохраняется в истории.
- Все права проверяются сервером.
- Telegram является основным каналом входа, но не единственной точкой доступа к системе.

## Требует уточнения

- Нужен ли вход по email через magic link или пароль.
- Как сотрудник впервые привязывает Telegram к своей учетной записи.
- Кто подтверждает привязку сотрудника к Telegram.

## Email как резервный канал

Email используется не только для резервного входа, но и для резервной доставки важных уведомлений, если Telegram недоступен.
